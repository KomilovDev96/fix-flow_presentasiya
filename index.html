Обзор системы

Назначение, цели, охват и ключевые характеристики продукта

FixFlow — это веб-ориентированная платформа класса Service Desk, разработанная для автоматизации процессов управления заявками на ремонт оборудования, технического обслуживания помещений, ИТ-поддержки и других видов сервисных запросов внутри крупных организаций с разветвлённой филиальной структурой.

🌟
Цель проекта
  • Сокращение времени реакции на инциденты
  • Централизованный учёт всех сервисных запросов
  • Повышение прозрачности выполнения работ
  • Контроль SLA и KPI исполнителей
🎯
Целевая аудитория
  • Торговые сети и ритейлеры
  • Банки и финансовые учреждения
  • Производственные предприятия
  • Управляющие компании (недвижимость)
⚡
Ключевые ценности
  • Мобильный доступ через Telegram Mini App
  • Real-time уведомления (WebSocket)
  • Dashboard-аналитика в стиле BI-систем
  • Полный audit log всех действий

Поля, валидация и бизнес-правила при создании заявки

Поля заявки

Поле Тип Обязательность Описание Валидация
object_id UUID / FK Обязательно Объект (ПБО / Филиал) Из справочника объектов
group_id UUID / FK Обязательно Группа работ (ремонт, электрика, IT…) Из справочника групп
category_id UUID / FK Обязательно Категория (подкатегория группы) Зависит от group_id
title VARCHAR(200) Обязательно Краткое название заявки 3–200 символов
description TEXT Обязательно Подробное описание проблемы Мин. 10 символов
priority ENUM Обязательно LOW / MEDIUM / HIGH / CRITICAL Одно из 4 значений
photos ARRAY<URL> Опционально Фотографии проблемы (до 5 шт.) JPG/PNG, макс. 5 МБ
location POINT (lat/lng) Опционально Геолокация из браузера / Telegram WGS-84 координаты
deadline TIMESTAMP Опционально Желаемый срок выполнения Не раньше текущей даты

Система уведомлений

Real-time WebSocket и Telegram-бот интеграция

💬
WebSocket (Real-time)
  • Подключение при открытии веб-приложения
  • Мгновенные уведомления без перезагрузки
  • Toast-уведомления в UI
  • Обновление счётчиков и статусов в реальном времени
  • Комнаты (rooms) по ролям: managers, executors
  • Библиотека: Socket.IO / NestJS Gateway
✈
Telegram Bot
  • Push-уведомления в Telegram-чат пользователя
  • Действия прямо из чата: /take, /done, /reject
  • Форматированные сообщения с деталями заявки
  • Deep link → переход в Mini App
  • Webhook-архитектура (не polling)
  • Использует telegramId из профиля пользователя
